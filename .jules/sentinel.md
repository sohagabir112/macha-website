## 2025-02-17 - Missing Authorization Checks (IDOR) in Server Actions
**Vulnerability:** Server actions interacting with user data, specifically modifying items via ID (e.g. `updateCartItem` and `removeCartItem`), lacked proper authorization checks. They accepted an `itemId` but did not verify if that item belonged to the authenticated user. This allows a malicious user to guess or enumerate IDs to modify or delete other users' data (Insecure Direct Object Reference).
**Learning:** In Next.js Server Actions, even if the action is called from an authenticated context in the client, the server must independently verify authentication (`supabase.auth.getUser()`) and verify that the user is authorized to perform the action on the specific resource.
**Prevention:** Always authenticate the user within the server action. Scope Supabase database queries to the authenticated user's ID by appending `.eq('user_id', user.id)` to all operations (Select, Update, Delete) involving user-owned data to prevent IDOR. Never trust the client to only send IDs that belong to the user.
