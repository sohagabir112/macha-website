## 2024-05-15 - [CRITICAL] Fix authorization bypass in cart API
**Vulnerability:** Server Actions in `app/cart/actions.ts` (`updateCartItem` and `removeCartItem`) did not verify if the cart item being updated or deleted belonged to the currently authenticated user. They only queried by `id`. This creates an Insecure Direct Object Reference (IDOR) vulnerability where any user could modify or delete another user's cart item by guessing its `id`.
**Learning:** In applications using Server Actions directly with a database (like Supabase), row-level security or explicit query scoping is required. Even if the UI doesn't expose other users' IDs, the server action is a public endpoint that can be called with arbitrary arguments.
**Prevention:** Always authenticate the user within the Server Action and explicitly scope all update/delete database queries to include `.eq('user_id', user.id)` alongside the object's `id` to ensure the user owns the resource they are trying to modify.
