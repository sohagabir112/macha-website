## 2024-05-24 - [IDOR in Server Actions]
**Vulnerability:** IDOR vulnerability was found in the `updateCartItem` and `removeCartItem` server actions in `app/cart/actions.ts`. The actions allowed updating or deleting records purely by using the `id` argument (which came from client data), without verifying that the logged-in user owned the records.
**Learning:** Next.js Server Actions don't inherently have context about the authenticated user unless you explicitly fetch the session from Supabase (e.g. `supabase.auth.getUser()`). Furthermore, even when a user is authenticated, queries modifying rows must explicitly scope their `.eq()`, `.update()`, or `.delete()` operations to that user's ID (`.eq('user_id', user.id)`) if they are interacting with user-specific data.
**Prevention:** Always authenticate users explicitly in server actions that modify database records. Scope database modifications securely by appending `.eq('user_id', user.id)` to prevent users from altering or deleting rows that do not belong to them. Avoid exposing unhandled database errors to the frontend.
